# Historia #2.3: Consulta de suscripciones vigentes del cliente

**Estado:** Borrador (PO)

---

## Historia de Usuario

**Como** cliente de la plataforma BTG Pactual,  
**Quiero** consultar el listado de mis suscripciones vigentes a fondos de inversión,  
**Para** conocer mi portafolio activo, ver cuánto tengo invertido en cada fondo y poder decidir cuáles cancelar si lo deseo.

## Descripción

El cliente puede consultar un listado con todas sus suscripciones que se encuentran en estado ACTIVO. Este listado le permite visualizar su portafolio de inversiones vigente, incluyendo el nombre del fondo, monto invertido, fecha de suscripción e identificador de cada suscripción. Esta consulta es esencial para que el cliente identifique cuál suscripción cancelar (por ID) en la funcionalidad de cancelación (HU 2.2).

---

## Criterios de Aceptación

### Escenario 1: Cliente con suscripciones activas

- **Dado** que un cliente tiene una o más suscripciones con estado ACTIVO
- **Cuando** consulta su listado de suscripciones vigentes
- **Entonces** se retorna únicamente las suscripciones con estado ACTIVO de ese cliente, excluyendo las CANCELADAS

### Escenario 2: Cliente sin suscripciones vigentes

- **Dado** que un cliente no tiene suscripciones con estado ACTIVO (nunca se suscribió o canceló todas)
- **Cuando** consulta su listado de suscripciones vigentes
- **Entonces** se retorna una lista vacía con código HTTP 200

### Escenario 3: Datos incluidos en cada suscripción del listado

- **Dado** que un cliente tiene suscripciones activas
- **Cuando** recibe el listado
- **Entonces** cada suscripción incluye: ID de suscripción, ID del fondo, nombre del fondo, monto invertido, fecha de suscripción y estado

### Escenario 4: Aislamiento de datos entre clientes

- **Dado** que existen suscripciones de múltiples clientes en el sistema
- **Cuando** un cliente consulta su listado
- **Entonces** solo se retornan las suscripciones que pertenecen a ese cliente específico (aislamiento por userId del token JWT, según HU 1.3)

---

## Información Recopilada

### Usuario y Contexto

- **Tipo de usuario:** Cliente registrado y autenticado en la plataforma (ver HU 1.1 y HU 1.2)
- **Permisos requeridos:** Rol CLIENTE — solo puede consultar sus propias suscripciones (ver HU 1.3, aislamiento de datos)
- **Valor de negocio:** Proporcionar transparencia al cliente sobre su portafolio activo de fondos de inversión, permitiéndole tomar decisiones informadas sobre cancelaciones o nuevas suscripciones

### Reglas de Negocio

- Solo retorna suscripciones con estado ACTIVO del cliente autenticado
- Si no tiene suscripciones activas, se retorna lista vacía (no error)
- El aislamiento de datos garantiza que un cliente nunca ve suscripciones de otro cliente
- El listado sirve como insumo para seleccionar cuál suscripción cancelar (HU 2.2)

### Datos Incluidos por Suscripción

| Campo | Descripción |
|---|---|
| suscripcionId | Identificador único de la suscripción |
| fondoId | Identificador del fondo |
| nombreFondo | Nombre del fondo de inversión |
| monto | Monto invertido en el fondo |
| fechaSuscripcion | Fecha y hora de la suscripción |
| estado | Estado de la suscripción (siempre ACTIVO en este listado) |

### Sistemas Externos

No aplica — es una consulta de solo lectura sobre datos internos.
